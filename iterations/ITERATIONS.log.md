# 审计工具迭代日志 (Audit Tool Iteration Log)

## [V1.0] - 基础规则筛查
- **目标**: 实现关键词、大额整数、节假日等基础特征识别。
- **状态**: 样本量过大 (8000+)，噪音极多。

## [V2.0] - 业务逻辑初步建模
- **目标**: 加入融资性贸易对冲、年底突击收入检测。
- **状态**: 具备初步审计穿透力，但未区分手工与自动流水。

## [V3.0] - SAP 结构降噪
- **目标**: 针对 SAP 凭证类型 (AF/SA/KD) 进行降噪，剔除折旧流水。
- **状态**: 样本降至 3000 笔，但仍受资金池和预提冲销干扰。

## [V4.0] - 深度数据挖掘与生命周期审计 (当前)
- **目标**: 引入预提冲销闭环、资金池语义屏蔽、用户行为画像、化整为零检测。
- **关键逻辑**: 穿透凭证看‘借贷对冲’，重点审计核心人员林翠池的手工调整凭证。
- **预期结果**: 样本量控制在 250 笔以内。
## [V4.1] - 逻辑修正与异常归类
- **目标**: 修正化整为零规则对大额流水的误报，增加大额重复流水识别。
- **关键改进**: 
  - 为化整为零增加金额上限 (<5万)。
  - 针对 >5000万 的大额重复流水，标记为‘大额重复’而非‘化整为零’，并降低常规划转权重。
  - 增加‘同名划转’到屏蔽库。
